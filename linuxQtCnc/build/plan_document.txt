第五章  关键接口迁移策略
5.1  NML 通信接口迁移
原来的 NML 通信在 Python 端通过 linuxcnc_util.py 封装。重构时将其迁移为 Qt C++ 类：
// 原 Python 接口:
s = linuxcnc.stat()   # 读取状态
c = linuxcnc.command() # 发送命令
e = linuxcnc.error()  # 读取错误
// 新 Qt/C++ 接口:
class LcncStat : public NmlReader {
  EMC_STAT* stat();
  void poll();  // 定时更新
};
class LcncCommand : public NmlWriter {
  void mode(LinuxcncMode);
  void home(int axis);
  void jog(int axis, double velocity);
  void mdi(const QString&amp; command);
};
5.2  HAL 接口迁移
原来的 Python HAL 接口 (hal.py) 迁移为 C++ RAII 封装：
// 原 Python 接口:
h = hal.component(&quot;mycomp&quot;)
h.newpin(&quot;out&quot;, hal.HAL_FLOAT, hal.HAL_OUT)
h.ready()
// 新 C++ 接口:
class HalComponent {
  HalComponent(const char* name);
  ~HalComponent(); // 自动 hal_exit
  HalPinFloat newPin(const char* name, HalDir dir);
  HalSignal link(const char* pin, const char* sig);
  void ready();
};
5.3  Canon 命令接口迁移
canon.hh 定义的规范命令接口已是 C++ 接口，无需修改。但需注意：
1. 原来的 GET_EXTERNAL_* 查询函数通过全局函数指针实现，重构时建议改为虚函数接口，便于测试
2. emccanon.cc 中的 canon 命令实现需保留，它是解释器到运动控制的桥梁
3. 移除 Python 相关的 pycall 和 interp_python 代码，用 C++ 回调替代

第七章  重构后架构总览

重构完成后，linuxQtCnc 将是一个纯 C/C++ + Qt6 的现代化数控系统，具备以下特点：
1. 统一技朠栈：全部代码为 C/C++ + Qt6，无 Python/Tcl/GTK 依赖
